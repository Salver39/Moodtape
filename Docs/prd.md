# Product Requirements Document (PRD) — Moodtape Telegram Bot

## Overview / Problems
Moodtape — это Telegram-бот, который создает музыкальные плейлисты по эмоциональному описанию пользователя. Он комбинирует субъективное описание настроения с музыкальными предпочтениями, извлечёнными из Spotify или Apple Music. Это решает проблему нецелевых, шаблонных рекомендаций в стримингах.

Проблемы, которые решает продукт:
- Пользователи не могут быстро найти музыку, соответствующую их текущему состоянию или атмосфере
- Стандартные алгоритмы рекомендаций игнорируют контекст (эмоции, образы, погода)
- Нет возможности гибко влиять на подбор без технических навыков или сложных интерфейсов

## Key User Flows
| Flow | Description |
|------|-------------|
| Start & Service Choice | Пользователь пишет `/start`, выбирает Spotify или Apple Music |
| Mood Description Input | Пишет эмоциональное описание (пример: «осень, одиночество и тёплый чай») |
| GPT-4o Parsing | Описание преобразуется в параметры аудио и жанров (valence, energy и др.) |
| Playlist Generation | Строится персональный плейлист с учётом лайков или предпочтений |
| Playlist Delivery | Ссылка на плейлист отправляется пользователю |
| Feedback Collection | Пользователь может оставить 👍/👎 или комментарий на плейлист |
| Personalization | Фидбэк учитывается в будущем подборе музыки |

## Functional Requirements
- Использование модели ChatGPT-4o для разбора текста и генерации музыкального запроса
- Интеграция с Spotify API: лайки, создание плейлистов
- Интеграция с Apple Music API: создание плейлистов, dev-токен
- Telegram-интерфейс с мультиязычной поддержкой (RU, EN, ES)
- Обработка текста произвольной формы и интерпретация эмоций
- Хранение фидбэков и их использование при следующих запросах
- Docker-среда, Railway-деплой, `.env` поддержка через `python-dotenv`

## Non-Goals
- Распознавание аудио по голосу/музыке
- Продвинутый ML-алгоритм персонализации (на старте — простая логика)
- Поддержка сторонних стримингов кроме Spotify/Apple
- UI/UX вне Telegram-интерфейса

## Milestones & Release Plan
| Milestone | Цель |
|-----------|------|
| M1 | Инициализация проекта, Dockerfile, структура каталогов, `.env` |
| M2 | Telegram-интерфейс: запуск, выбор стриминга, ввод текста |
| M3 | Интеграция GPT-4o и генерация параметров плейлиста |
| M4 | Поддержка Spotify: авторизация, лайки, создание плейлиста |
| M5 | Поддержка Apple Music: генерация плейлистов |
| M6 | Хранение и учёт фидбэка пользователя |
| M7 | Готовый MVP, деплой на Railway, тестирование, открытие доступа |
